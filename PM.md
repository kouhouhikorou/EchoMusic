## 1. 项目概述

### 1.1 产品定位

EchoMusic 是一款**开源的跨平台音乐播放器**，兼顾网易云音乐的核心体验与本地播放器的自由度。用户既可以从本地导入音乐文件，也可以通过接入多平台音乐镜像 API（网易云 / QQ / 酷狗等）实现在线搜索、试听、歌词获取等功能。内置**独立账号系统**，用户数据完全由自己掌控，不依赖任何音乐平台的账号体系，杜绝封号风险。

### 1.2 核心价值主张

| 维度 | 说明 |
|------|------|
| **跨平台** | 同一套代码，Web / PWA / 后续可选 APK 封装 |
| **双模播放** | 本地文件播放 + 在线流媒体搜索播放 |
| **多音源聚合** | 统一搜索入口，聚合网易云/QQ/酷狗等多个音乐平台 |
| **独立账号体系** | 自建账号系统，数据云端同步，不依赖第三方平台账号 |
| **完整音乐体验** | 封面、歌词滚动、收藏、歌单、下载 |
| **PWA 优先** | 离线缓存、后台播放、系统媒体控制，移动端原生级体验 |
| **开源可部署** | 发布到 GitHub，前端 + 后端均开源，支持一键自部署 |

### 1.3 竞品参考

- **网易云音乐** — UI/UX 风格参考，歌单、每日推荐等社区氛围
- **YesPlayMusic** — 开源的第三方网易云播放器，PWA 实现出色，Vue 3 技术栈
- **Listen1** — 多平台聚合播放器，浏览器扩展形态
- **lx-music (洛雪音乐助手)** — 多音源聚合桌面端，Electron 架构
- **Resso** — 歌词滚动交互优秀，逐字高亮 + 社交歌词

## 2. 目标用户与使用场景

### 2.1 目标用户画像

| 画像 | 描述 |
|------|------|
| **音乐爱好者** | 喜欢收藏歌曲、看歌词、浏览封面，希望有沉浸式体验 |
| **开发者/技术用户** | 愿意自行部署，喜欢开源项目 |
| **多设备用户** | 在电脑和手机上切换使用，期望数据同步 |
| **隐私敏感用户** | 不想使用大厂 App 的臃肿功能，偏好轻量播放器 |

### 2.2 典型使用场景

1. **桌面端** — 浏览器打开 Web 版，或安装 PWA 到桌面，搜索、播放、看歌词
2. **手机端** — 安装 PWA 到主屏幕 / 使用 WebView 封装的 APK，锁屏后台播放，通知栏控制
3. **离线场景** — PWA Service Worker 缓存已播放资源，无网络时播放已缓存歌曲
4. **本地导入** — 把本地 MP3/FLAC 文件拖入播放器，自动解析元数据

### 3.2 播放器核心功能详情

#### 3.2.1 后台播放 (移动端 — 关键)

针对移动端后台播放的完整方案：

- **Media Session API** — 注册播放元数据 (标题/歌手/封面)，使系统通知栏/锁屏显示当前歌曲信息并支持上/下曲、播放/暂停
- **AudioContext / `<audio>` 持久化** — 确保页面进入后台时音频不中断
- **Service Worker** — 缓存关键资源，使 PWA 安装后可在离线状态启动应用

#### 3.2.2 歌词滚动

- 支持 **LRC 格式** 标准歌词解析（`[mm:ss.xx]text`）
- 支持 **逐字歌词 (YRC / 增强 LRC)** 的逐字高亮
- 当前播放句**高亮 + 居中**
- 手动滑动浏览，松手后 N 秒自动回到当前播放位置
- 支持歌词偏移调整（快/慢，±0.5s 步进）

#### 3.2.3 封面展示

- 播放页：全宽正方形封面 + 毛玻璃背景
- 迷你播放器：小尺寸圆角封面缩略图
- 列表：封面缩略图 + 歌曲信息
- 无封面时的 fallback：根据主色调生成渐变默认图
- 唱片旋转动画 (可配置开关)

#### 3.2.4 音源故障自动切换

当当前音源播放失败时（404 / 超时 / 403 等），系统自动执行以下流程：

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│  1. 检测播放失败                                     │
│     ↓                                               │
│  2. Toast 提醒 (底部弹出，3s 自动消失)                 │
│     ┌──────────────────────────────────────┐        │
│     │ ⚠ 网易云音源暂不可用 (404)              │        │
│     │   正在尝试切换到 QQ 音源...             │        │
│     └──────────────────────────────────────┘        │
│     ↓                                               │
│  3. 按优先级自动尝试下一音源                          │
│     优先级: 网易云 → QQ → 酷狗 → Bilibili             │
│     ↓                                               │
│  4a. 成功 → Toast "已切换至 QQ 音源 ✓" (1.5s)        │
│  4b. 全部失败 → 弹窗 "所有音源均不可用，请稍后重试"     │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**故障类型与提示文案**：

| 错误类型 | Toast 提示 |
|------|------|
| 404 (资源不存在) | `⚠ {音源名} 音源暂不可用 (404)，正在切换到 {下一音源}...` |
| 超时 (>8s) | `⚠ {音源名} 响应超时，正在切换到 {下一音源}...` |
| 403 (区域限制) | `⚠ {音源名} 暂不支持当前地区访问，正在切换到 {下一音源}...` |
| 成功切换 | `✓ 已切换至 {音源名}` |
| 全部失败 | `✗ 所有音源均不可用，请稍后重试` (Modal) |

**配置项**（设置页）：
- 音源尝试优先级 (拖拽排序)
- 切换前是否弹窗确认 (开关，默认仅 Toast 提醒)
- 超时时间 (默认 8s)

### 5.2 后端服务 (自建)

EchoMusic 的后端服务分为两个独立模块：

#### 5.2.1 多音源聚合 API (音乐数据)

聚合多个音乐平台的镜像 API，统一输出格式。前端通过 `source` 参数切换音源。

| 音源 | 底层 API | 说明 |
|------|------|------|
| **网易云音乐** | [NeteaseCloudMusicApi](https://github.com/Binaryify/NeteaseCloudMusicApi) | 最成熟，歌单/评论/推荐丰富 |
| **QQ 音乐** | [QMusicApi](https://github.com/) 或自封装 | 曲库大，部分无损音质 |
| **酷狗音乐** | 自封装 / 开源项目 | 补充曲库 |
| **Bilibili** | [bilibili-API-collect](https://github.com/SocialSisterYi/bilibili-API-collect) | 大量 MV / 翻唱 / 现场 / ACG 音源，视频音频分离播放 |

**统一聚合接口设计**：

```typescript
// 搜索 (聚合多平台)
GET /api/search?keyword=xxx&source=netease|qq|kugou&type=song|album|artist|playlist

// 获取播放 URL
GET /api/song/url?id=xxx&source=netease&quality=standard|higher|lossless

// 获取歌词
GET /api/lyric?id=xxx&source=netease

// 统一响应格式
interface SearchResponse {
  source: 'netease' | 'qq' | 'kugou';
  songs: Song[];
  total: number;
  hasMore: boolean;
}
```

**部署方案**：
- **方案 A (推荐)** — 项目提供公共 API 服务 (部署在 Vercel / Railway 等免费平台)，用户开箱即用
- **方案 B** — 用户自行部署 (提供 Docker 一键部署 + 详细文档)，数据完全自主
- 前端内置「API 地址」配置，支持一键切换公共 API / 自部署 API

> ⚠️ **公共 API 免责声明**：公共 API 仅供学习和测试使用，不保证可用性和稳定性。建议重度用户自行部署。

#### 5.2.2 账号 & 数据同步服务

独立的账号体系，与任何音乐平台无关：

| 功能 | 接口 | 说明 |
|------|------|------|
| 注册 | `POST /auth/register` | 邮箱/用户名 + 密码注册 |
| 登录 | `POST /auth/login` | JWT Token 认证 |
| Token 刷新 | `POST /auth/refresh` | 无感刷新 |
| 同步收藏 | `POST /sync/favorites` | 增量同步红心歌曲 |
| 同步歌单 | `POST /sync/playlists` | 增量同步歌单结构 |
| 同步播放记录 | `POST /sync/history` | 跨设备继续播放 |
| 数据导出 | `GET /sync/export` | 全量 JSON 导出 |

**安全设计**：
- 密码 bcrypt 哈希存储
- JWT + Refresh Token 双 token 机制
- 不存储任何第三方平台 Cookie/Token
- 与音乐平台的 API Key 完全隔离
- 用户数据仅在自建数据库中，不流经任何音乐平台

- 设置页 (`/settings`) 底部保留「查看免责声明」入口，可随时重新查看

**设置页底部常驻入口**：
```
设置 > 关于 > 「使用须知与免责声明」 (可点击查看完整文本)
```

### 5.3 PWA 配置

```yaml
PWA 能力清单:
  - installable: 可添加到桌面
  - offline: Service Worker 缓存静态资源 + 已播放音频
  - background_sync: 后台同步收藏/播放记录
  - notifications: 系统级播放通知
  - media_session: 锁屏/通知栏播放控制
  - share_target: 支持从其他应用分享歌曲链接
```

### 5.4 移动端方案 (PWA 优先)

**核心策略：PWA 优先，按需封装**

**第一阶段 — PWA (必做，Phase 2)**

用户在手机浏览器打开 → 添加到主屏幕 → 获得接近原生体验：
- 独立窗口运行 (无浏览器地址栏)
- 系统级 Media Session (锁屏封面 + 控制)
- Service Worker 离线缓存
- Web Share Target (接收其他 App 分享)
- 推送通知 (可选)

**PWA vs 原生能力对比**：

| 能力 | PWA (iOS) | PWA (Android) | 说明 |
|------|------|------|------|
| 添加到桌面 | ✅ 16.4+ | ✅ | |
| 后台音频播放 | ✅ | ✅ | iOS 需用户先交互一次 |
| 锁屏控制 | ✅ | ✅ | Media Session API |
| 离线使用 | ✅ | ✅ | Service Worker |
| 通知 | ⚠️ 有限 | ✅ | iOS 18+ 支持 Web Push |
| 后台同步 | ⚠️ | ✅ | Background Sync API |
| 文件系统访问 | ⚠️ | ✅ | File System Access API |

**第二阶段 — Capacitor 封装 (评估后决定，Phase 5+)**

如果 PWA 体验在 iOS 有明显短板 (如后台播放不稳定/通知不达)，再用 Capacitor 封装为 APK/IPA：
- 同一套 Vue 代码，零改动打包
- 原生 Audio Session 更稳定
- 应用商店分发

### 6.2 核心数据模型

```typescript
// 歌曲
interface Song {
  id: string | number;
  title: string;
  artist: string;
  album: string;
  cover: string;          // 封面图片 URL
  url: string;            // 播放地址
  duration: number;       // 时长 (秒)
  lyric: string;          // LRC 歌词文本
  tlyric?: string;        // 翻译歌词
  source: 'local' | 'cloud';
  localPath?: string;     // 本地文件路径 (local 模式)
  format?: string;        // 音频格式
}

// 歌单
interface Playlist {
  id: string;
  name: string;
  cover: string;
  description: string;
  songs: Song[];
  created: number;
  updated: number;
}

// 播放器状态
interface PlayerState {
  currentSong: Song | null;
  queue: Song[];          // 播放队列
  history: Song[];        // 播放历史 (最近 N 首)
  index: number;          // 当前队列索引
  isPlaying: boolean;
  mode: 'list' | 'single' | 'shuffle';
  volume: number;
  progress: number;
  duration: number;
}

// 用户数据
interface UserData {
  favorites: Song[];        // 红心歌曲
  playlists: Playlist[];    // 用户歌单
  recentPlays: Song[];      // 最近播放
  downloadQueue: Download[];// 下载队列
  settings: Settings;
}
```

## 8. 非功能性需求

### 8.1 性能

| 指标 | 目标 |
|------|------|
| **首次加载 (FCP)** | < 2s (桌面) / < 3s (移动) |
| **搜索响应** | < 500ms (含 API 请求) |
| **音频切换延迟** | < 300ms |
| **PWA 安装大小** | < 5MB (不含缓存音频) |
| **内存占用** | < 150MB (含音频解码) |

### 8.2 兼容性

| 平台 | 目标 |
|------|------|
| **Chrome / Edge** | 最新 2 个大版本 |
| **Safari** | 16+ (支持 PWA manifest) |
| **Firefox** | 最新 2 个大版本 |
| **Android WebView / Chrome** | 最新 2 个大版本 |
| **iOS Safari (PWA)** | 16.4+ |

### 8.3 安全性

- API 请求数据仅存储在用户本地，不上传至任何第三方服务器
- API 地址由用户自行配置，不内置硬编码的密钥
- HTTPS 传输 (部署时强制)
- XSS 防护 (DOMPurify 或框架默认转义)
